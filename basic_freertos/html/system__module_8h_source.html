347/* System Module Status Routine Pointer
348
349 Function:
350 SYS_STATUS (* SYS_MODULE_STATUS_ROUTINE) ( SYS_MODULE_OBJ object )
351
352 Summary:
353 Pointer to a routine that gets the current status of a system module
354 (driver, library, or system-maintained application).
355
356 Description:
357 This data type is a pointer to a routine that gets the current status of a
358 system module (driver, library, or system-maintained application).
359
360 Precondition:
361 The low-level board initialization must have (and will be) completed and the
362 module's initialization routine will have been called before the system will
363 call the status routine for any modules.
364
365 Parameters:
366 object - Handle to the module instance
367
368 Returns:
369 One of the possible status codes from SYS_STATUS
370
371 Example:
372 None.
373
374 Remarks:
375 A module's status operation can be used to determine when any of the
376 other module level operations has completed as well as to obtain general
377 status of the module. The value returned by the status routine will be
378 checked after calling any of the module operations to find out when they
379 have completed.
380
381 If the status operation returns SYS_STATUS_BUSY, the previous operation
382 has not yet completed. Once the status operation returns SYS_STATUS_READY,
383 any previous operations have completed.
384
385 The value of SYS_STATUS_ERROR is negative (-1). A module may define
386 module-specific error values of less or equal SYS_STATUS_ERROR_EXTENDED
387 (-10).
388
389 The status function must NEVER block.
390
391 If the status operation returns an error value, the error may be cleared by
392 calling the reinitialize operation. If that fails, the deinitialize
393 operation will need to be called, followed by the initialize operation to
394 return to normal operations.
395*/
396
397typedef SYS_STATUS (* SYS_MODULE_STATUS_ROUTINE) ( SYS_MODULE_OBJ object );
